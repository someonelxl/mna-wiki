# Source / Raw Source 模板文档 v1.0

## 1. 文档定位
本文件用于定义本项目中 Source / Raw Source 相关对象的标准骨架与最小分工。
它不负责解释项目为什么存在，也不负责定义系统整体结构，而是回答：
- raw source 应该是什么
- source record 应该是什么
- 两者如何分层
- AI 和人应如何围绕这两层协同工作

本文件属于模板层文档，应受《项目总需求定位 v2.0》《系统设计文档 v1.0》《Rulebook 初稿 v0.1》约束。

## 2. Source / Raw Source 模板的作用
Source / Raw Source 模板的作用有五个：
1. 固定来源层和知识层的边界
2. 让原始资料能够稳定进入系统
3. 让来源可以被追溯、挂接和引用
4. 让 AI 在 ingest 时有清晰的输入对象
5. 防止把 raw source、source record、wiki 页面混写在一起

## 3. 两层结构的基本定义
### 3.1 Raw Source
Raw Source 是原始资料的文本化承载层。
它属于 `raw-sources/`，是 source of truth 的可读层之一。
它的任务是：
- 保存原始可读资料
- 尽量保持原始性
- 为 ingest 提供输入
- 为后续判断提供证据基础

### 3.2 Source Record
Source Record 是来源记录页。
它属于 Wiki 侧的记录层对象，不等于原始资料本身。
它的任务是：
- 记录这个来源是什么
- 记录它来自哪里
- 记录它与哪些 Case / Concept 有关
- 记录它的资料性质和核心价值
- 作为从 Wiki 回到来源层的入口

## 4. 设计原则
### 4.1 原始资料与整理记录必须分开
Raw Source 不等于 Source Record。
Raw Source 负责保存原始文本层，Source Record 负责保存解释性索引和挂接信息。

### 4.2 Source Record 不承载全文
Source Record 的职责不是复制来源全文，而是：
- 记录
- 索引
- 概括
- 挂接
- 指向

### 4.3 Raw Source 优先追求原始性与可追溯性
进入 raw-sources 的资料，优先追求：
- 能回到原始出处
- 能被 AI 稳定读取
- 不被总结性改写污染

### 4.4 AI 先整理记录层，人审核关键挂接
AI 可以先整理 Source Record 的首版，但：
- 来源性质
- 挂接对象
- 是否值得正式进入系统
- 是否影响某个 Concept / Case
这些关键判断应由人审核。

### 4.5 来源层应服务知识增长，而不是变成收藏夹
系统引入来源，不是为了囤积材料，而是为了：
- 支撑判断
- 推动 ingest
- 形成连接
- 长出新母题和新角度

## 5. Raw Source 的标准骨架
当前阶段，Raw Source 至少应满足以下 6 个部分或属性。

## 5.1 基本身份信息
### 作用
让系统知道这份 raw source 是什么。
### 至少应明确的信息
- 来源标题
- 来源类型
- 原始出处
- 导入时间
- 当前保存形式

### 原则
这一部分优先追求：
- 可识别
- 可追踪
- 可回溯

---

## 5.2 原始文本层
### 作用
承载真正可读、可 ingest 的文本内容。
### 原则
这部分应尽量保持：
- 原始性
- 完整性
- 不被总结性语言污染

### 注意
这里不是摘要区，也不是评论区。

---

## 5.3 来源元信息
### 作用
记录来源的基本背景。
### 可包含的信息
- 作者 / 发布方
- 发布时间
- 链接
- 文件来源
- 平台信息
- 文件格式

### 原则
不是所有字段都必须有，但应尽量保留后续可能有用的最基本溯源信息。

---

## 5.4 来源分类
### 作用
帮助系统理解这份来源属于哪类资料。
### 常见分类
- web
- pdf
- transcript
- book
- report
- paper
- filing
- announcement

### 原则
优先做粗分类，不追求一开始极细。

---

## 5.5 当前归属状态
### 作用
标记这份 raw source 当前处于什么阶段。
### 可能的状态
- inbox
- imported
- pending-ingest
- linked
- archived

### 原则
这一部分是为了后续流程管理，不是为了展示内容。

---

## 5.6 原始文件与文本层关系说明
### 作用
说明当前 raw source 与原始文件或原始链接的关系。
### 常见情况
- 网页链接 + Markdown 剪藏
- PDF 原件 + 文本提取层
- 视频链接 + transcript 文本层
- 书籍文件 + 拆分文本层

### 原则
应能让人或 AI 明白：
- 当前看到的是哪一层
- 原始文件是否还存在
- 从哪里可以回到原始出处

## 6. Source Record 的标准骨架
当前阶段，Source Record 至少应包含以下 8 个部分。

## 6.1 Source 基本信息
### 作用
快速识别这个来源是什么。
### 建议承载的信息
- Source ID
- 来源标题
- 来源类型
- 发布时间
- 发布方 / 作者
- 当前状态

### 原则
优先清楚、稳定、可引用。

---

## 6.2 原始位置
### 作用
回答：
**这份来源最原始的出处在哪里。**
### 可能包括
- 原始网页链接
- 原始 PDF 文件位置
- 视频链接
- 外部文件路径
- 归档路径

### 原则
必须保证后续可回去找原始资料。

---

## 6.3 当前系统位置
### 作用
回答：
**这份来源在系统里的哪一层、哪个路径。**
### 可能包括
- 对应 raw source 文件路径
- 对应归档目录
- 是否在 inbox / cases / general

### 原则
让来源的系统内位置清楚可查。

---

## 6.4 来源性质说明
### 作用
说明这份来源是什么类型、有什么资料性质。
### 可写的内容
- 财报 / 年报
- 新闻 / 报道
- 公告 / filing
- 访谈 / transcript
- 书籍章节
- 研报 / 文献
- 一手资料 / 二手资料

### 原则
这一部分优先服务于后续判断来源可信度和用途。

---

## 6.5 核心要点
### 作用
提炼这份来源最值得保留的点。
### 原则
不是全文摘要，而是回答：
- 这份来源为什么值得进系统
- 它支持什么判断
- 它最有价值的信息是什么

### 注意
当前阶段保持轻量，不写成长篇分析页。

---

## 6.6 关联对象
### 作用
说明这份来源当前主要挂接到哪些对象。
### 可能包括
- 相关 Case
- 相关 Concept
- 以后可能扩展到 analysis / summary

### 原则
至少要回答：
- 这份来源目前主要服务谁
- 它是 case-specific 还是 general source
- 是否影响多个对象

---

## 6.7 使用建议 / 后续动作
### 作用
说明这份来源接下来适合怎么用。
### 可能包括
- 待 ingest
- 待补 Concept
- 待关联 Case
- 待回读
- 待核实
- 适合内容提炼

### 原则
这一部分帮助系统和人判断下一步，而不是做最终结论。

---

## 6.8 备注 / 风险提示
### 作用
记录这份来源需要额外注意的问题。
### 可能包括
- 来源可信度一般
- 链接可能失效
- 文本提取质量不高
- 时间较旧
- 与其他来源存在冲突
- 需要人工复核

### 原则
这一部分是为了降低误用风险。

## 7. 当前阶段暂不纳入核心骨架的部分
为了控制模板重量，以下内容当前阶段不作为必填项：
- 全文逐段摘要
- 证据句逐条抽取
- 复杂可信度打分
- superseded / deprecated 完整状态系统
- 全量引用摘录区
- 自动对比其他来源的冲突报告
- 复杂 provenance 树

这些内容以后可以扩展，但当前阶段先不纳入核心骨架。

## 8. AI 与人的协作方式
### 8.1 AI 负责什么
AI 在 Source / Raw Source 处理中，优先负责：
- 判断来源类型
- 生成 Source Record 首版
- 提炼核心要点草稿
- 推荐可能关联的 Case / Concept
- 提醒可能的风险和后续动作
- 协助把来源转成可读的 raw source 文本层

### 8.2 人负责什么
人优先负责：
- 判断这份来源值不值得正式进入系统
- 审核关联对象是否成立
- 判断其对母题和案例的实际价值
- 处理提取质量差、来源模糊或边界不清的情况
- 决定是否写回到正式知识层

### 8.3 当前阶段的默认原则
AI 优先整理和建议，人负责挂接和取舍。

## 9. 当前阶段的最小完成标准
一个 Source / Raw Source 在当前阶段，如果满足以下条件，可视为达到“最小可用”状态：
1. 已有可读的 raw source 文本层或明确原始出处
2. 已有基本身份信息
3. 已有来源性质说明
4. 已有核心要点
5. 已有至少一个关联对象或待关联对象
6. 已有系统内位置
7. 已能从 Source Record 回到 raw source 或原始出处

## 10. 本模板文档的作用
本文件用于为后续：
- 来源进入系统
- raw source 文本层归档
- Source Record 创建
- ingest 输入准备
- review / lint 检查

提供统一骨架。
未来如果 Source / Raw Source 结构需要升级，应优先更新本文件，而不是让来源层对象自由长成不同风格。

## 11. 当前结论
当前阶段的 Source / Raw Source 模板核心可以概括为：
- 先把原始资料存对层
- 再把来源记录写清楚
- 再把它和 Case / Concept 连起来
- 再让它进入 ingest 和知识增长流程

只要这条骨架立住，来源层就已经开始为整个 Wiki 提供稳定输入。
