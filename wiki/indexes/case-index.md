# Case Index v0.1

## 页面定位
本页是当前阶段的 Case 主入口页。
它用于组织和浏览当前正式纳入系统的案例对象，帮助用户快速识别案例并进入单案例页面。
本页是入口页，不替代 Case 页面本身。

## 页面职责
本页负责：
- 提供 Case 的统一入口
- 提供最小案例识别信息
- 支持从案例进入单案例研究页
- 为后续按交易结构、行业、地区、状态扩展预留位置

本页不负责：
- 重复案例正文
- 代替 Case 页面进行完整分析
- 承载过重比较内容
- 承担规则定义功能

## 当前收录范围
当前阶段，本页收录：
- 已正式建立的 Case 页面
- 已具备最小可用状态的案例对象
- 编号、标题、基本识别信息已明确的案例

当前阶段暂不收录：
- 尚未正式建页的候选案例
- 仅存在于讨论中的临时案例
- 尚无最小来源支撑的案例对象

## 使用说明
建议使用方式：
1. 先看 `case_id`
2. 再看案例名称
3. 再看交易结构和基本识别信息
4. 如需分析，请进入对应 Case 页面

## 当前列表字段
当前阶段，列表建议至少包含：
- `case_id`
- `case_name`
- `transaction_structure`
- `industry`
- `region`
- `status`
- 可选的一句话定义
